# 데일리 브리핑 파이프라인

오늘 날짜를 Asia/Seoul 기준으로 확인한다. `_posts/`에 오늘 자 글(`YYYY-MM-DD-*.md`)이
이미 있으면 **아무것도 하지 않고 중단**한다 (중복 발행 방지).

아래 4단계를 순서대로 실행한다. 각 단계는 이전 단계의 산출물에 의존하므로 병렬 실행 금지.

## 1. 리서치

`researcher` 서브에이전트를 실행한다. 프롬프트에 포함할 것:
- 오늘 날짜
- 결과 저장 경로: `scratch/research-YYYY-MM-DD.md` (scratch/ 폴더가 없으면 생성)

완료 후 스크래치 파일이 존재하고 항목이 3건 이상인지 확인한다.
3건 미만이면 파이프라인을 중단하고 4단계의 실패 처리로 간다.

## 2. 작성

`writer` 서브에이전트를 실행한다. 프롬프트에 포함할 것:
- 리서치 파일 경로
- 출력 경로: `_posts/YYYY-MM-DD-daily-brief.md`

## 3. 검수

`editor` 서브에이전트를 실행한다. 프롬프트에 포함할 것:
- 초안 경로와 리서치 파일 경로
- 오늘 날짜

편집자가 "발행 불가"를 보고하면 4단계의 실패 처리로 간다.

## 4. 발행

성공 시:
1. `memory/covered.md` 맨 위에 오늘 섹션을 추가하고, 다룬 주제를 한 줄씩 기록한다
   (형식: `- 주제 키워드 — 한 줄 요지`). 30일 이전 항목은 삭제해 파일을 짧게 유지한다.
2. `scratch/` 안의 오늘 파일을 삭제한다.
3. `git add` 후 커밋: `post: YYYY-MM-DD 데일리 브리핑` → `git push`.
4. 사용자에게 발행된 글 제목과 헤드라인 목록을 요약 보고한다.

실패 시 (리서치 부족, 발행 불가 판정 등):
- **커밋하지 않는다.** 만들다 만 `_posts/` 초안이 있으면 삭제한다.
- `memory/notes.md`에 날짜와 실패 사유를 기록하고, 이것만 커밋·푸시한다
  (커밋 메시지: `note: YYYY-MM-DD 발행 실패 기록`).
