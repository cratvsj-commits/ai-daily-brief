## 2026-07-23 발행 실패

리서치(7건)·작성 단계까지는 정상 완료했으나, 검수(editor) 단계에서 WebFetch가 이 세션의
모든 외부 URL 요청에 예외 없이 403을 반환해 팩트체크를 수행할 수 없었다. 대조군으로 시도한
완전 무관한 도메인(google.com, en.wikipedia.org, example.com)도 동일하게 403이 발생해,
특정 소스의 문제가 아니라 세션 전체의 조직 egress 정책에 의한 차단으로 확인됐다
(`/root/.ccr/README.md` 진단 가이드 기준 "403/407 from the proxy" 케이스).
가장 파급력이 큰 헤드라인(OpenAI 모델의 허깅페이스 해킹 관련)을 1차 소스로 대조 검증하지
못해 편집자가 "발행 불가" 판정. 미완성 초안(`_posts/2026-07-23-daily-brief.md`)은 삭제했고
커밋하지 않았다. 리서치 스크래치 파일(`scratch/research-2026-07-23.md`)은 다음 실행 시
재검수용으로 남겨둠. 다음 실행 시 WebFetch/프록시 접근이 정상화됐는지 먼저 확인 필요.
